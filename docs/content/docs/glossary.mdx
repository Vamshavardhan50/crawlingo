# Glossary

## A

**Auto-match** — A self-healing system that recovers broken CSS selectors by comparing DOM fingerprints. Uses Jaro-Winkler and Jaccard similarity scoring to find elements when the original CSS selector fails.

## B

**Browser profile** — A TLS fingerprint configuration that mimics a specific browser (Chrome, Firefox, Safari) to avoid bot detection.

## C

**Change detection** — The process of comparing old and new extracted values to identify changes. Used by Watch to detect price changes, stock changes, and content changes.

**ChangeEvent** — An object passed to watch callbacks containing details about a detected change (field name, old value, new value, change type).

**Crawl** — A multi-page concurrent crawler that discovers pages via link following, extracts fields from each page, and supports webhooks and scheduling.

**CrawlResults** — The output of a Python `Crawl.build()`, containing multiple `DatasetResult` objects with export methods.

**CSS selector** — A pattern used to select DOM elements. Crawlingo supports tag, class, ID, and compound selectors.

## D

**Dataset** — A structured data extractor for a single URL. Define fields with selectors and receive a `DatasetResult` with export capabilities.

**DatasetResult** — The output of a `Dataset.build()` containing extracted field values with export methods (JSON, CSV, Parquet, Pandas DataFrame).

**DOM fingerprint** — A structured profile of a DOM element (tag, text, classes, attributes, ancestor path) used for auto-match self-healing.

**DomTree** — The parsed HTML document stored as a flat contiguous `Vec<DomNode>` for cache-friendly traversal.

## E

**Element** — A single DOM node with text, HTML, and attribute access. Python SDK adds navigation methods (parent, children, siblings).

**ElementCollection** — A collection of DOM nodes with aggregate operations (first, last, nth, text, attr, filter, map).

## F

**Fetcher** — The HTTP client layer that executes requests with configurable strategies (standard or stealthy).

**Fetcher tier** — The mode of HTTP client: `"standard"` for basic HTTP operations, `"stealthy"` for TLS fingerprint emulation.

**Fingerprint store** — A sled database that persists DOM fingerprints for auto-match recovery across sessions.

## H

**HostRateLimiter** — Per-host token bucket rate limiter using the `governor` crate. Blocks until a token is available.

## L

**lol-html** — Library of Love HTML rewriter used for fast streaming HTML parsing.

## M

**MCP** — Model Context Protocol. Crawlingo's built-in SSE server lets LLM agents run structured crawls and extract data.

**memchr** — A SIMD-accelerated byte-level search library used for text anchor queries.

## P

**Page** — A fetched web page with a parsed immutable DOM tree and selector query methods.

## R

**Rate limiting** — Per-host token bucket algorithm that limits requests per second to avoid overwhelming target servers.

## S

**Selector engine** — Unified query interface dispatching to CSS, XPath, regex, text anchor, after_text, and before_text.

**Session** — Shared configuration container for cookies, headers, proxy, rate limits, and auto-match settings.

**Stealth mode** — Fetcher mode that emulates browser TLS fingerprints to avoid bot detection.

## T

**Text anchor** — A search method (`find_text`, `after_text`, `before_text`) that uses SIMD-accelerated byte search to locate elements by text content.

## W

**Watch** — A periodic page monitor that polls a URL, re-extracts fields, detects changes, and fires callbacks.
