## The Fragile Selector Problem

In traditional web scraping, scripts rely on static identifiers like `id` attributes, CSS classes, or absolute XPath routes. When site developers update their code, rename selectors for CSS modules, or restructure their layout, those selectors instantly break.

This causes scrapers to fail silently or throw exceptions, leading to broken data pipelines and requiring human intervention to find the new selector, rewrite the code, and redeploy the script.

## Quick Example

Set up a session, crawl pages with structured extraction, and export data:

<Tabs items={['Python', 'Node.js']}>
  <Tab value="Python">

    ```python
    from crawlingo import Session

    session = Session()
    session.auto_match(True)
    session.timeout(30)

    crawl = session.crawl("https://news.ycombinator.com")
    crawl.follow(".morelink")
    crawl.limit(5)
    crawl.field("title", ".titleline > a")
    crawl.field("points", ".score")

    results = crawl.build()
    print(f"Extracted {len(results)} pages.")
    results.to_json("hn_posts.json")
    results.to_csv("hn_posts.csv")
    ```

  </Tab>
  <Tab value="Node.js">

    ```typescript
    import { Session, Crawl } from 'crawlingo';

    const session = new Session();
    session.autoMatch(true);
    session.timeout(30);

    const crawl = new Crawl("https://news.ycombinator.com", session);
    crawl.follow(".morelink");
    crawl.limit(5);
    crawl.field("title", ".titleline > a");
    crawl.field("points", ".score");

    const results = await crawl.run();
    console.log(`Extracted ${results.length} pages.`);
    ```

  </Tab>
</Tabs>
