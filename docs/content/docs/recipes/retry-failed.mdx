# Retry Failed Pages

## Problem

Some pages fail with network errors or timeouts. You need to retry with different settings.

## Solution

Implement a retry wrapper that catches errors, rotates proxies, and retries with backoff.

<Tabs items={['Python', 'Node.js']}>
  <Tab value="Python">

    ```python
    from crawlingo import Session, Page
    import time, random

    def fetch_with_retry(url: str, session: Session, max_retries=3):
        for attempt in range(max_retries):
            try:
                page = Page(url, session=session, timeout=15)
                return page
            except Exception as e:
                if attempt == max_retries - 1:
                    raise

                wait = (2 ** attempt) + random.uniform(0, 1)
                print(f"Attempt {attempt + 1} failed: {e}. Retrying in {wait:.1f}s")

                # Rotate proxy on retry
                if "proxy" in str(e).lower():
                    session.proxy_pool(["http://backup-proxy:8080"])

                time.sleep(wait)

        return None
    ```

  </Tab>
  <Tab value="Node.js">

    ```typescript
    import { Session, Page } from 'crawlingo';

    async function fetchWithRetry(url: string, session: Session, maxRetries = 3) {
      for (let attempt = 0; attempt < maxRetries; attempt++) {
        try {
          const page = await Page.create(url, { session, timeout: 15 });
          return page;
        } catch (e) {
          if (attempt === maxRetries - 1) throw e;
          const wait = Math.pow(2, attempt) + Math.random();
          console.log(`Attempt ${attempt + 1} failed: ${e}. Retrying in ${wait.toFixed(1)}s`);
          await new Promise(r => setTimeout(r, wait * 1000));
        }
      }
    }
    ```

  </Tab>
</Tabs>

## Best Practices

- Exponential backoff (2^attempt seconds) prevents hammering servers
- Add jitter (±random) to avoid thundering herd
- Rotate proxies on consecutive failures
- Fail fast on 4xx errors (client errors won't resolve with retries)
- Use timeout=15 for quick failure detection
