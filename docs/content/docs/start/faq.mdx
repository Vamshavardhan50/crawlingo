# FAQ

## General

### What is Crawlingo?

Crawlingo is a Rust-powered web scraping framework with self-healing selectors, multi-language SDKs (Python, Node.js, Rust, Go), and built-in change detection. See [Why Crawlingo](/why-crawlingo).

### How is it different from Scrapy or BeautifulSoup?

Crawlingo's key differentiator is **auto-match self-healing** — it uses DOM fingerprinting and fuzzy matching to recover broken selectors when websites update. It also has a Rust core for performance, built-in stealth TLS, an MCP server for LLM integration, and native multi-format export. See the [comparison table](/why-crawlingo#key-differentiators).

### Do I need to know Rust to use Crawlingo?

No. The Rust core is compiled into native binaries that ship with the Python and Node.js SDKs. You write Python or TypeScript code.

### Is Crawlingo free?

Yes, Crawlingo is open source under the MIT license.

## Installation

### pip install fails

Ensure Python 3.9+ is installed. Try `pip install --only-binary crawlingo`. If no pre-built wheel exists for your platform, install Rust and build from source: `pip install crawlingo --no-binary crawlingo`.

### npm install fails

Ensure Node.js 16+. Pre-built `.node` binaries exist for Windows, macOS, and Linux (x86_64, aarch64). If your platform is unsupported, file an issue.

### Which platforms are supported?

Windows, macOS, and Linux on x86_64 and aarch64 architectures.

### How do I extract structured data?

Use `Dataset` for single-page extraction or `Crawl` for multi-page. See [First Dataset](/first-dataset) and [First Crawl](/first-crawl).

### How do I follow links during a crawl?

Use `crawl.follow("a.next-page")` with a CSS selector that matches the links to follow. See [Your First Crawl](/first-crawl#step-3-configure-link-following).

### How do I monitor page changes?

Use `Watch`. Define fields to monitor and callbacks for change events:

<Tabs items={['Python', 'Node.js']}>
  <Tab value="Python">

    ```python
    watch = Watch("https://example.com/product/1")
    watch.field("price", "span.price")
    watch.interval(300)
    watch.on_price_change(lambda ev: print(f"Price: {ev.new_value}"))
    watch.run()
    ```

  </Tab>
  <Tab value="Node.js">

    ```typescript
    const watch = new Watch("https://example.com/product/1");
    watch.field("price", "span.price");
    watch.interval(300);
    watch.run((err, event) => {
      if (event) console.log(`Price: ${event.newValue}`);
    });
    ```

  </Tab>
</Tabs>

## Performance

### How many pages can I crawl per second?

This depends on network conditions, target server speed, and concurrency settings. With 10 concurrent workers and a fast target, expect 50–100 pages per minute. See [Performance Optimization](/best-practices#performance).

### Is Crawlingo memory efficient?

Yes. The DOM tree is stored as a flat `Vec<DomNode>`, not a tree of pointers. Results stream directly to disk via Arrow RecordBatch without materializing large in-memory structures.

### Does it support async?

Python has `build_async()` on Dataset. Node.js is fully async/promise-based. The Rust core uses Tokio for async I/O.

## Troubleshooting

### CSS selector returns empty results

1. Verify the element exists: `page.html()` to inspect
2. Enable auto-match: `auto_match(True)`
3. Try XPath or text anchor as alternative
4. Check if the element is loaded dynamically (SPA)

### Crawl finds few or no pages

1. Verify the `follow` selector matches links on the start page
2. Increase `depth` and `limit`
3. Check if the target site blocks automated requests

### Auto-match fails

The page structure may have changed significantly. Update your selectors, or lower the similarity threshold by adjusting weights with `auto_match_weights()`.

### Rate limiting errors

Reduce `rate_limit()` value, increase crawl `delay()`, or rotate proxies.

## Concepts

### How does auto-match work?

It records a `DomFingerprint` (tag, text, classes, attributes, ancestor path) on successful selector matches. When a selector fails, it compares the current DOM against stored fingerprints using Jaro-Winkler + Jaccard scoring, and returns the best match above a 0.5 threshold. See [Auto-Matcher](/concepts#auto-matcher).

### What is a fingerprint store?

A sled database that persists DOM fingerprints. It is lazily opened on first access and cached in the Session. Use `fingerprint_path()` to control where it is stored.

### Can I export to formats other than JSON/CSV/Parquet?

Currently JSON, CSV, and Parquet are supported. Parquet uses Apache Arrow RecordBatch internally. CSV follows RFC 4180.

## SDK Differences

### Why does Node.js use `.run()` instead of `.build()`?

Python uses `.build()` which is synchronous but releases the GIL during fetch. Node.js is fully async and uses `.run()` which returns a `Promise<DatasetResult[]>`.

### Why doesn't Node.js have Element navigation methods?

Node.js Element only supports `.text`, `.html`, and `.attr(name)`. Navigation (`.parent()`, `.children()`, etc.) is Python-only due to the NAPI-RS binding surface.

### Why doesn't Node.js have hooks?

Hooks (before_fetch, after_fetch, etc.) are Python-only. Use session-level configuration for equivalent functionality.
