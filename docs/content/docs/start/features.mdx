## CSS Selectors

Query element lists using standard tag names, IDs, class paths, and hierarchy filters. Built-in caching avoids redundant page parsing.

<Tabs items={['Python', 'Node.js']}>
  <Tab value="Python">
    ```python
    page = session.page("https://example.com")

    # Query tags
    headings = page.css("h1")

    # Class match
    cards = page.css(".product-card")

    # ID match
    header = page.css("#main-header")
    ```
  </Tab>
  <Tab value="Node.js">
    ```typescript
    import { Page } from 'crawlingo';

    const page = await Page.create("https://example.com");

    const headings = page.css("h1");
    const cards = page.css(".product-card");
    ```
  </Tab>
</Tabs>

## Text Anchor (SIMD-Accelerated)

Anchor elements based on literal content values. Excellent for extracting variable data located next to static titles (e.g. price tags or sku names).

<Tabs items={['Python', 'Node.js']}>
  <Tab value="Python">
    ```python
    element = page.find_text("SKU Number:")
    detail = page.after_text("SKU Number:")
    ```
  </Tab>
  <Tab value="Node.js">
    ```typescript
    const element = page.findText("SKU Number:");
    const detail = page.afterText("SKU Number:");
    ```
  </Tab>
</Tabs>

## Auto-Match Self-Healing

## Change Monitoring Watches

Setup polling on elements to trigger callbacks when elements are added, deleted, or values (like price changes) drift.

<Tabs items={['Python', 'Node.js']}>
  <Tab value="Python">
    ```python
    watch = session.watch("https://example.com/item")
    watch.field("price", ".price")
    watch.interval(60)
    watch.on_price_change(lambda ev: print(f"Price modified: {ev.old_value} -> {ev.new_value}"))
    watch.run()
    ```
  </Tab>
  <Tab value="Node.js">
    ```typescript
    const watch = new Watch("https://example.com/item", session);
    watch.field("price", ".price");
    watch.interval(60);
    watch.run((err, event) => {
      if (event) console.log(`Price modified: ${event.oldValue} -> ${event.newValue}`);
    });
    ```
  </Tab>
</Tabs>
