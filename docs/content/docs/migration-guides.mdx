## Migrating from Other Frameworks

### From BeautifulSoup

| BeautifulSoup | Crawlingo |
|--------------|-----------|
| `soup.find("tag")` | `page.css("tag").first()` |
| `soup.find_all("tag")` | `page.css("tag")` |
| `soup.select(".class")` | `page.css(".class")` |
| `soup.select_one("#id")` | `page.css("#id").first()` |
| `tag.text` | `el.text()` / `el.text` |
| `tag["attr"]` | `el.attr("attr")` |
| `tag.parent` | `el.parent()` |
| `tag.find_next_sibling()` | `el.next()` |

**Key differences:**
- Crawlingo uses lazy element creation — `page.css()` returns indices, not Element objects
- Element objects are materialized on `.first()`, `.nth()`, `[index]`
- Add `Session` for shared configuration across requests

### From Scrapy

| Scrapy | Crawlingo |
|--------|-----------|
| `scrapy.Spider` | `Crawl` class |
| `start_urls` | URL passed to constructor |
| `parse()` callback | Pre/post-processing hooks |
| `scrapy.Request()` | Automatic in `Crawl.follow()` |
| `response.css()` | `page.css()` |
| `Item` classes | `Dataset.field()` definitions |
| `Feed exports` | `.to_json()`, `.to_csv()`, `.to_parquet()` |
| `DOWNLOAD_DELAY` | `crawl.delay()` or `session.rate_limit()` |
| `CONCURRENT_REQUESTS` | `crawl.concurrency()` |

**Key differences:**
- Crawlingo is callback-free — define fields and run
- Auto-match eliminates manual selector maintenance
- Progress is synchronous (`.build()` returns when done)
- Built-in change detection via `Watch`
- Built-in stealth TLS (no middleware needed)

## Breaking Changes

No breaking changes yet. Crawlingo is pre-1.0 and APIs may change between minor versions.

## Deprecation Notices

None currently.

## Future Compatibility

SDK versioning follows the Rust core version. Major versions (1.x, 2.x) may change FFI boundaries. Minor versions (1.1, 1.2) add features without breaking existing code.
